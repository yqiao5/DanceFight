Running in a Sandbox

Windows (tested with Sandboxie):

  • Due to a lack of permissions, Rewired is unable to receive device information from Raw Input when run inside a sandbox. Joysticks will not work properly in Raw Input mode. (As a note, Unity itself is also unable to receive this information, rendering joystick support in Unity broken under the default security settings.)
  • Direct Input mode is able to receive joystick input, but joystick plug/unplug notifications are blocked, so hot-plugging does not work properly even in Direct Input mode.

Building from Command Line

In order to provide native input for various platforms, Rewired implements a system where platform dependencies are set at the time the platform mode is changed in the editor. This allows Rewired to include only the necessary libraries for a specific platform to conserve memory in game builds. When building from a command line, this dependency swapping is broken because Unity starts building before Rewired is able to switch libraries. Rewiredâ€™s method for switching the current dependencies will not work with command line building in the current implementation. Alterate methods will be explored and support for command line building may be possible in the future, but at present, it does not work properly.

Building from the command line does work, but switching platforms during command line building does not. So a workaround for this issue is to open the project in the Unity editor, change the current platform to your desired build target, exit, then run your command line build. Since the platform has already been changed in the editor and the data files updated to include the build target's dependencies, command line builds to this build target will succeed. You can run the command line build as many times as necessary as long as you don't specify a different build target.

 

Windows App Store support not working in Unity 5

Due to a new bug introduced in Unity 5, Rewired is unable to be compiled in this version of Unity. The bug relates to classes that inherit from other classes contained within a DLL that contain serialized data. I have made Unity aware of this bug, but there is no possible workaround for Rewired without major structural changes to the Rewired Input Manager data storage system that would result in data loss for all users. This is unacceptable, so as of Unity 5, until Unity fixes their bug, Rewired no longer supports Windows app store. Unity 4.x branch is unaffected and will work as usual. Recommendation: If you require Windows App Store support, do not upgrade to Unity 5 at this time.
